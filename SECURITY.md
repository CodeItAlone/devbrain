# Security Policy

## Supported Versions

Only the latest major or minor releases of DevBrain are actively supported with security updates. We recommend all users keep their installations up to date.

| Version | Supported |
|---|---|
| **v0.1.x** | ✅ Yes |
| **< v0.1.0** | ❌ No |

---

## Reporting a Vulnerability

We take the security of DevBrain seriously. If you discover a security vulnerability, **please do not report it publicly through standard GitHub Issues.**

Instead, please report it via one of the following secure channels:

1.  **GitHub Security Advisory**: Go to the "Security" tab on the GitHub repository page, click on "Advisories", and choose "Report a vulnerability". This opens a private draft advisory where you can share details safely with the maintainers.
2.  **Maintainer Contact**: If private advisories are unavailable, please send a detailed email to `security@devbrain.dev`.

### What to Include in a Report:
*   A detailed description of the vulnerability.
*   Step-by-step instructions or a minimal proof-of-concept (PoC) script to reproduce the issue.
*   Potential impact of the vulnerability (e.g. unauthorized local file reads, arbitrary command injection).
*   Any suggestions for remediation or patches.

---

## Our Process

*   **Acknowledgement**: We will acknowledge receipt of your report within 48 hours.
*   **Assessment**: We will investigate and assess the severity of the issue within 5 business days, keeping you updated on our progress.
*   **Remediation**: If validated, we will work on a patch and coordinate a public release with a credit to the researcher (unless anonymity is requested).
